"Qui est là ?"

On désire développer une structure qui permet de savoir qui entre et sort de l'entreprise.

Le projet comporte deux outils :

  1. Une application sera disponible sur une tablette à l'accueil.
  2. Un tableau de bord sécurisé par login/pass sera disponible sur desktop.

1. APPLICATION

Entrée d'une personne

Quand une personne rentre, elle choisi sur l'application l'onglet "Entrée" et doit saisir :

  • son nom
  • son prénom
  • son email
  • l'objet de sa visite via un menu déroulant :
    • soit un membre du personnel à visiter
    • soit une formation donnée ce jour

L'application génère une étiquette avec :

  • son nom
  • son prénom
  • le local où se rendre
  • le nom de la personne visitée ou l'intitulé de la formation.
  • L'ID du visiteur
  • En version 2 : Un QR code est généré qui reprend l'ID du visiteur (unique).

Sortie d'une personne

Quand une personne sort, elle choisi l'onglet "sortie" et encode son ID.
(En version 2, elle scanne son QR code).
Elle reçoit un message de confirmation, ce qui enregistre sa sortie.

Retour d'une personne

Si une personne revient dans le bâtiment et qu'elle a son ID, elle l'encode. (en version 2 elle scanne son QR code). Le système affiche ses coordonnées. Elle les confirme et doit choisir entre une visite ou une formation.

Si elle n'a pas son ID (En version2 : son QR code), elle peut faire une recherche sur son email.

2. ADMIN (Tableau de bord)

Pour tous les utilisateurs connectés :

  • La partie admin permettra de visiualiser en live qui est dans le bâtiment.
  • Le système permettra de parcourir l'historique (liste des visites par jour choisi dans un calendrier ou saisi dans un input).

Pour certains rôles d'utilisateurs connectés, le système permettra de gérer :

  • les membres du personnel (nom, prénom, fonction [formateur ou administration], local et n° de tél. interne)
  • les formations (Intitlué, date, local, formateur).

Structure attendue

Le choix est libre mais :

  • Développement d'une API et d'une application pour l'accueil.
  • Rendu SSR (template avec rendu côté serveur, PHP traditionnel en somme) accepté pour la partie Admin.
